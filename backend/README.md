# Pipeline Monitoring Demo — Backend

FastAPI backend for the semi-live pipeline corridor monitoring demo.

## What this is

A small API that powers a geospatial web app demonstrating pipeline
monitoring capabilities using satellite data. Built around the
**IOCL Salaya-Mathura crude pipeline** (1,870 km).

Currently uses **simulated scan results** — the output format is
identical to what real Sentinel-1/Sentinel-2 processing would produce,
so the frontend and deploy path work unchanged when you swap in real data.

## Project structure

```
backend/
├── main.py                  # FastAPI app (serve this)
├── prep_pipeline_data.py    # One-time: build pipeline + buffers + tiles
├── run_full_scan.py         # One-time: produce cached scan results
├── requirements.txt
└── data/                    # Generated by the two scripts above
    ├── pipeline.geojson
    ├── buffer_1km.geojson
    ├── buffer_5km.geojson
    ├── aoi_tiles.geojson
    ├── sample_anomalies.geojson
    ├── full_scan_results.geojson
    └── scan_metadata.json
```

## Setup (local)

```bash
pip install -r requirements.txt

# 1. Build baseline geospatial data (run once)
python prep_pipeline_data.py

# 2. Run the simulated full-corridor scan (run once, or whenever you
#    want a fresh baseline)
python run_full_scan.py

# 3. Start the API
uvicorn main:app --reload --port 8000
```

Visit http://localhost:8000 to see the service banner.
Visit http://localhost:8000/docs for the auto-generated Swagger UI.

## Endpoints

| Method | Path | Purpose | Latency |
|--------|------|---------|---------|
| GET    | `/api/baseline` | Pipeline geometry, buffers, analysis tiles | instant |
| GET    | `/api/full_scan` | Cached anomaly results (current state) | instant |
| GET    | `/api/scan_metadata` | Headline stats + tile summaries | instant |
| POST   | `/api/refresh` | Simulated full re-scan | 3–5 sec |
| GET    | `/api/health` | Health check | instant |

### `POST /api/refresh` behaviour

- Waits 3–5 seconds (simulating compute)
- Applies subtle jitter to the cached results:
  - Timestamps bump forward
  - 1–3 confidence scores nudge
  - 1 severity may flip (medium ↔ high)
  - Occasionally adds 1 new anomaly or removes 1 low-severity one
- Returns updated anomaly set and headline stats
- Mutates the in-memory cache, so subsequent `GET /api/full_scan`
  calls reflect the new state

This gives the frontend a believable "live" refresh while staying
deterministic, fast, and deployable on free tiers.

## Deploy

### Railway / Render (free tier)

1. Push this backend folder to a GitHub repo
2. Create a new service, connect the repo
3. Build command: `pip install -r requirements.txt`
4. Start command: `uvicorn main:app --host 0.0.0.0 --port $PORT`
5. Make sure the `data/` folder is committed (it's needed at startup)

### CORS

Currently open (`*`) for demo convenience. Before sharing with clients,
edit `main.py` and restrict `allow_origins` to your frontend domain.

## Swapping in real Sentinel data later

The simulation lives in a single function: `analyze_tile()` inside
`run_full_scan.py`. Its signature is:

```python
def analyze_tile(tile_feature, scan_time) -> list[dict]:
    # returns list of GeoJSON Feature dicts (Point anomalies)
```

Replace the body with real Copernicus Data Space Ecosystem calls:
1. Query STAC API for latest Sentinel-1 GRD + Sentinel-2 L2A scenes
   intersecting `tile_feature.geometry`
2. Run SAR coherence change detection (via SNAP, or `snappy`/`pyroSAR`)
3. Run NDVI difference on Sentinel-2
4. Threshold + cluster pixels, optional vision-model classification
5. Emit GeoJSON Points with the same property schema

Nothing else in the stack needs to change.
